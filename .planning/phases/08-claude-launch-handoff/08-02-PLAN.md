<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| standalone CLI -> Claude child process | The CLI injects sensitive local gateway env into a spawned external executable. |
| user-provided argv -> launch helper | Arbitrary Claude args cross from operator input into process launch and must remain unchanged without shell injection risk. |
| prepared runtime/manifest -> launch env | Client token and local gateway URL move from workspace state into Claude’s environment. |
| launch failure -> operator guidance | Missing-command and spawn failures must not look like successful session startup. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-02-01 | T | `standalone-cli/src/launch/claude.ts` argv handling | mitigate | Use direct `spawn('claude', args, { shell: false })` with the original args array so passthrough stays unchanged and shell injection is avoided. |
| T-08-02-02 | I | launch env construction | mitigate | Build the child env in one helper, inject only the exact four legacy vars, and keep output/help/test surfaces secret-safe with no raw token rendering. |
| T-08-02-03 | D | missing-command and spawn-error path | mitigate | Map `ENOENT` and related launch failures to explicit install/PATH guidance and treat them as hard failures with no success banner. |
| T-08-02-04 | E | launch-surface implementation scope | mitigate | Keep all Phase 8 code inside `standalone-cli/`, reuse `prepareRuntimeEnvironment()`, and forbid launcher-file generation or shell-wrapper fallbacks that would bypass the isolated package boundary. |
</threat_model>
