<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| caller `cwd` -> repo-root gateway build/spawn | Untrusted invocation directories must not redirect build or runtime commands away from the intended repo root. |
| stored runtime metadata -> local process control | Manifest/runtime data influences whether the CLI signals a PID or starts a fresh runtime. |
| standalone CLI workspace -> filesystem | Local artifact paths cross from package code into the operator filesystem and must not escape through symlinks. |
| health probe network path -> readiness decision | A single hung local socket must not stall the entire readiness and launch pipeline. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01-01 | T | `standalone-cli/src/environment/runtime.ts` repo-root resolution | mitigate | Resolve the repo root from a package-owned anchor and verify build/spawn paths against that root before launching the gateway. |
| T-08-01-02 | D | `standalone-cli/src/environment/runtime.ts` health polling | mitigate | Enforce per-request timeout handling on each `/_health` probe and fail with bounded actionable diagnostics instead of hanging indefinitely. |
| T-08-01-03 | E | stale PID shutdown path in `standalone-cli/src/environment/runtime.ts` | mitigate | Require standalone-cli-owned runtime evidence before signaling a PID, and clear stale metadata rather than killing an unverified process. |
| T-08-01-04 | T | `standalone-cli/src/environment/workspace.ts` path validation | mitigate | Use symlink-aware realpath checks so manifest/config/runtime files cannot escape the CLI-owned workspace through symlink traversal. |
