<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| local machine credential stores -> standalone CLI | Sensitive OAuth material crosses from Keychain or a local credentials file into the CLI process. |
| standalone CLI -> operator output | Discovery results must remain actionable without leaking raw access or refresh tokens. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-02-01 | I | `standalone-cli/src/output.ts` and CLI rendering | mitigate | Render source names, status, and guidance only; never print raw `accessToken` or `refreshToken` values in success or failure output. |
| T-06-02-02 | T | `standalone-cli/src/credential-discovery/discover.ts` | mitigate | Preserve the exact ordered fallback from the legacy scripts and cover it with fixture-driven tests so later edits cannot silently reorder sources. |
| T-06-02-03 | D | Keychain and credentials-file adapters | mitigate | Distinguish “not found”, malformed JSON, and missing `refreshToken` failures so operators get actionable guidance instead of ambiguous exits. |
| T-06-02-04 | E | package implementation boundary | mitigate | Keep the implementation inside `standalone-cli/` and forbid imports from protected legacy paths or stale `dist/standalone-cli/` artifacts. |
</threat_model>
