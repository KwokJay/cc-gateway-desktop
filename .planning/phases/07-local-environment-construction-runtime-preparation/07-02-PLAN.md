<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| parent shell env -> runtime child process | Proxy settings and other inherited env values cross from the operator shell into the spawned gateway runtime. |
| standalone CLI -> local runtime process | The CLI is responsible for starting or reusing the local gateway process without falsely reporting readiness. |
| runtime metadata -> rerun reuse logic | Stored pid/config fingerprint data influences whether the CLI reuses or restarts a process on rerun. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-02-01 | I | `standalone-cli/src/environment/proxy-env.ts` and child env handling | mitigate | Preserve the exact existing proxy-variable precedence and pass through the inherited keys unchanged so proxy-aware outbound traffic cannot silently bypass operator proxy settings. |
| T-07-02-02 | D | `standalone-cli/src/environment/runtime.ts` | mitigate | Treat readiness as `/_health` success only, with timeout failure and actionable diagnostics, rather than process-spawn success or fixed sleeps. |
| T-07-02-03 | T | runtime reuse metadata | mitigate | Reuse only when stored runtime metadata matches the current manifest/config fingerprint and the runtime is still healthy; otherwise restart deterministically. |
| T-07-02-04 | E | `standalone-cli/src/cli.ts` command surface | mitigate | Keep the command surface limited to `prepare-runtime`; reject `claude` execution and argument passthrough until Phase 8 to avoid premature privilege or scope expansion. |
</threat_model>
