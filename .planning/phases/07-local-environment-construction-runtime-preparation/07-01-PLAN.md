<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| credential discovery result -> bootstrap manifest/config | Sensitive OAuth data crosses from Phase 06 discovery into persisted local bootstrap state. |
| standalone CLI workspace -> filesystem | The new CLI writes generated artifacts and must not escape into repo-root legacy paths or arbitrary filesystem targets. |
| rerun logic -> existing manifest | A malformed or duplicated manifest update could corrupt client auth or canonical identity state on repeat runs. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01-01 | I | `standalone-cli/src/environment/manifest.ts` and `config-render.ts` | mitigate | Persist OAuth values only in the CLI-owned workspace, keep output rendering secret-safe, and avoid printing raw tokens in bootstrap summaries or tests. |
| T-07-01-02 | T | `standalone-cli/src/environment/workspace.ts` | mitigate | Normalize all artifact paths under `~/.ccgw/standalone-cli/` and reject writes outside the resolved workspace root so Phase 7 cannot recreate repo-root `config.yaml` or `clients/` side effects. |
| T-07-01-03 | T | `standalone-cli/src/environment/bootstrap.ts` | mitigate | Use manifest-driven full config re-rendering instead of append-style YAML edits so repeat runs cannot duplicate `auth.tokens` entries or corrupt identity state. |
| T-07-01-04 | E | bootstrap artifact generation | mitigate | Keep implementation inside `standalone-cli/` and forbid imports or writes into protected legacy paths listed in the Phase 05 isolation boundary. |
</threat_model>
